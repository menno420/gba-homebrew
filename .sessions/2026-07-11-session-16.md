## Guard recipe (for the next slice)

**The py-desmume boot-poweroff trap:** the binding's core reads
`REG_KEYINPUT` as active-low ZERO (all buttons held) until the first
keypad latch; the BlocksDS default ARM7 (`sys/arm7/main_core/source/
main.c`) exits its loop on SELECT+START+L+R → libnds powers off at frame
1 with `SYSTEM POWERED OFF VIA ARM7 SPI POWER DEVICE` and a blank
framebuffer. Fix is `emu.input.keypad_update(0)` BEFORE the first
`cycle()` — already in `tools/nds-headless-check.py`; do not remove it,
and any new NDS harness must replicate it. Also: SDL needs
`SDL_VIDEODRIVER=dummy` headlessly (the tool sets it itself).
Slice-3 anchor points that remain unproven and were NOT claimed here:
memory-watch/telemetry reads via `emu.memory` (API exists —
`read_long`/`register_exec` verified importable, used for the magic-word
bisect — but no ELF-symbol resolution tool yet) and touch input
(`emu.input.touch_set_pos` exists, untested).
