## 💡 Session idea

A proof that claims an upgrade "makes the ship faster" is weakest as a
tier-word assert and strongest as a CONTROL-RUN PAIR: proofs 8 and 9
replay the identical recorded story and press the identical menu keys,
differing in ONE cursor row (REPAIR vs SAILS) — so both next duels latch
the SAME seed (728) with the SAME idle inputs, and the sails-II hull is
provably 110 fixed-point units ahead ten frames in (t5 34130 vs 34240),
growing exactly as the mirror integrates it. Rule extracted: **when a
purchase changes dynamics, spend one extra proof making the previous
proof its control run — same seed, same inputs, one variable.** The
cheap way to get that is scheduling every port story off the same
recorded bank route so the post-port duel always latches the same seed.
