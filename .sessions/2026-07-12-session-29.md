this cycle's status that a reader might act on (PR #61 "parked
READY+green") names a PR number, a head SHA, and a check count — so
this session could re-verify it in two API calls before restating it,
and any future reader can catch it going stale the same way. The
useful rule generalizes: when a status file asserts another artifact's
state, pin the assertion to the artifact's own coordinates (number +
SHA + count), never to prose alone — an unpinned "all green" can't be
cheaply audited and therefore won't be.
