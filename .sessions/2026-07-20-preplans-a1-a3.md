## 💡 Session idea

**A menu option should ship with its slice ladder attached, not a slice
*count*.** The #207 B1 skip and this A1/A3 write split on one line: B1 named
its five cuts (a ladder), so a pre-plan was padding; A1/A3 named only "~6
slices" (a count), so the ladder had to be invented at pick time — exactly the
latency a "make it instantly executable" pre-plan removes. The count "~6
slices" is a *promise about effort* that a picker cannot act on; the ladder is
the *executable object*. A menu that carried each open option's ladder inline
(as B1 did) would need no separate pre-plan pass at all — the pre-plan doc is
the debt interest on a menu written as pitch + estimate instead of pitch +
ladder. Next standing-menu author: enumerate, don't estimate.
