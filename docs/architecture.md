## Namespace protection — two mechanisms, both required

Two separate mechanisms guard the namespace, and they catch different
failure classes:

1. **A registry for runtime string identities** — event names, command
   names, settings keys, and any other string that selects behavior at
   runtime. Collisions here are invisible to static analysis.
2. **A static AST pass for Python symbol shadowing** — a later top-level
   `def` / `class` with the same name silently shadows the earlier one, and
   no import fails.

Neither mechanism subsumes the other. The registry cannot see symbol
shadowing; the AST pass cannot see string-keyed dispatch. Do not delete one
believing the other covers it.
