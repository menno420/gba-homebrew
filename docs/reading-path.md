## 0 · The standing authorization

Read-only cross-repo access is standing-authorized for every session in
this repo — do not burn turns re-deriving whether you may look.

- Readable siblings, and the private/dark exceptions (never read, never
  guess about a dark repo): every fleet sibling is public and raw-readable EXCEPT pokemon-mod-lab (PRIVATE/dark — never read it, never guess about its contents; raw fetch 404s it)
- Sanctioned read mechanics for a public sibling (no attach step needed):
  raw-content fetch
  (`https://raw.githubusercontent.com/<owner>/<repo>/<branch>/<path>`) ·
  `git clone` · `git ls-remote` (HEAD pins). Set a real User-Agent on
  scripted fetches — some hosts 403 a bare library default.
- **Boundaries that stand:** repo-API (MCP) calls stay scoped to the
  session's attached repo(s); **writes stay in this repo** — cross-repo
  work routes through the coordination bus (`control/README.md`), never
  direct pushes to a sibling; attach + clone remains the path for
  deep/audit work in a sibling.

## 1 · The one command

Orient on the whole fleet in one step before asking "what is the fleet
doing?" — it covers Tiers 1–2 below:

## 3 · Tiered depth (spend turns by task depth)

- **Tier 0 — this-repo-only task:** the boot set in the working agreement.
  No cross-repo reads needed.
- **Tier 1 — "what's the fleet doing?":** the one command above (plus the
  owner queue, if your fleet keeps one).
- **Tier 2 — acting on or with a sibling:** that sibling's heartbeat
  (`control/status.md`) in full, its ⚑ owner-ask blocks, its
  `docs/CAPABILITIES.md` / walls file, its open PRs.
- **Tier 3 — deep dive:** exactly the files a heartbeat points at (specs,
  plans, blueprints) — fetch those, not the tree. Write/audit depth:
  attach + clone.

## 4 · Truth rules (carry into every cross-repo read)

1. **Heartbeats are dated snapshots** — the `updated:` line tells you how
   stale; verify load-bearing claims at HEAD (`git ls-remote`, a fresh raw
   fetch) before acting (PL-006).
2. **Frozen archives mislead** — a parked/archived sibling's status file
   is history, not state; the sibling map above names where its truth
   moved.
3. **One writer per file** — never edit a sibling's control-bus files from
   here: its inbox is manager-written, its status is that seat's
   coordinator-only. Route cross-repo work as an ORDER through the bus.
4. **⚑ blocks are the owner interface** — read them before inventing an
   owner-ask.

Every reference this path hands you (a repo name, a heartbeat pointer, a
truth file) is resolved before you act on it — method: the
`chase-references` skill (see `docs/SKILLS.md`).

## 5 · Why this exists

A live session once burned ~3 turns re-discovering that (a) its fleet was
readable, (b) raw reads were the sanctioned mechanism, and (c) which files
carried each seat's truth — knowledge every prior doc implied but no
orientation surface stated as a route. This doc is that route. If it saves
you those turns it worked; if you find it stale, correct it in place —
orientation is first-class work.
